
* 메일서버 (SMTP Server)를 이용해 메일 전송
* SMTP는 Simple Mail Transfer Protocol 전자 메일 전송을 위한 표준 프로토콜 - 이메일을 송신하는 서버
* 메일서버 (SMTP Server) 를 통해 메일을 전송할때 두가지 방법이 있음

1) 메일 서버가 릴레이(relay)를 허용하는 경우
* 릴레이(Relay) : 외부(외부 네트워크) 에서 해당 메일서버(smtp서버)를 경유해서 외부로 메일을 보내는 것을 의미
- 계정이 존재하지 않더라도 메일 발송 가능
- 릴레이 허용하는 경우 잘못된 방식 (타인에 의한 스팸메일)로 악용 될 수 있음
- 특정 IP에서만 릴레이가 되도록 하는게 일반적


2) 메일 서버에 계정 인증 후 메일 전송하는 방법



1. 메일 계정 보안 설정
- SMTP Server를 통해 메일 전송 기능을 구현하기 위해 사용하고자 하는 메일의 SMTP 보안 단계를 낮춰야함

1-1. 구글
- [보안] - 보안 수준이 낮은 앱의 액세스 - 액세스 사용 설정 - 보안 수준이 낮은 앱 허용 '사용'으로 변경


2. mail 라이브러리 추가
- org.springframework.boot:spring-boot-starter-mail 추가


3. 메일을 보내기 위한 서버 정보를 프로퍼티에 등록
- spring boot를 사용하면 JavaMailSender Bean을 생성할 필요 없이, 자동으로 생성
